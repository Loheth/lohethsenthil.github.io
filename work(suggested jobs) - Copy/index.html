Problem Area

There is no mechanism to monitor the file status and notify the customer/ project owner about the batch files being transferred successfully.With this new scope, we intend to propose that our batch publisher library can monitor the file status currently created in the Databricks workspace and notify the consumer whether files are successfully transferred.


Scope of the Design:

  • With this new implementation, the batch publisher library will monitor a "Batch File Publisher Metastore" where all the files that are being created by the Databricks Job will push the data about the file being created.
  • The Batch publisher library will monitor the new files being created and will monitor till the file status changes to either "Delivered" or"Failed".
  • Once the status is finalized, it will send an email to end consumers.
  • The data present in the Batch File Publish Metastore will be continuously monitored by File Publish Compare & Store to store the data in Common Metadata Store
 

Proposal:

The idea is to push the metadata in the Batch File Publish Metastore where all the new files which are created by any Databricks jobs will be pushed. The Batch File Publisher will pick all the new files and will notify the aDDS to transfer them to the external SFTP system. Also, the Batch Publisher will monitor the existing that are still in Pending state to know whether the files are successfully transferred or not. The Batch File Publisher will also provide the facility to retry the failed files automatically three times.

A Batch File Publish Metastore will be created in the storage account with the above columns

Detailed Overview:

• Databricks Job will create the required aggregated file and will push the metadata in the Batch File Publisher Metastore without the delivered time, status. The Retry count will be set to zero
• The Batch Publisher will constantly monitor the metastore table and will check if the new files are being created and data is present for the same in the metastore.
The Status will be set as Started.
• Along, with it, there will be another method in the batch publisher that will constantly monitor the metastore for the files which are in "Started, Running, Pending" Status to check for their status. Also, it will check for the files which are in "Failed" status and check for the retry count is less than 3.
• Once, the files are in Delivered or Failed Status (Retry count = 3), it will send an email to the end consumers notifying the File Delivery Status.
